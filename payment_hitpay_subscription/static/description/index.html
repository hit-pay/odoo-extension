HitPay Subscription Payment Provider

Secure recurring subscription payments for Odoo 19 using HitPay with payment tokenization, automatic renewals, and seamless integration with Odoo Subscriptions.

Overview

HitPay Subscription Payment Provider integrates HitPay Recurring Billing with Odoo Subscriptions, allowing customers to securely authorize a payment method during the initial checkout. Future subscription renewals are charged automatically using the saved payment method without requiring customer interaction.

The module automatically manages payment tokenization, recurring billing, webhook registration, payment method lifecycle, and recurring subscription payments while remaining fully integrated with the standard Odoo payment framework.

Supported Recurring Payment Methods

Merchants can enable only the recurring payment methods supported by their HitPay account. The enabled payment methods are displayed automatically during checkout.

  • • Visa
  • • Mastercard
  • • American Express
  • • GrabPay
  • • ShopeePay
  • • Touch 'n Go

Configuration

  • 1. Install the HitPay Subscription Payment Provider module.
  • 2. Open Accounting → Configuration → Payment Providers.
  • 3. Configure your HitPay API credentials.
  • 4. Publish the payment provider.
  • 5. Enable the recurring payment methods supported by your HitPay account.
  • 6. The webhook is created automatically during module installation.

Subscription Checkout Flow

  • 1. Customer selects HitPay Subscription during checkout.
  • 2. Customer is redirected to the secure HitPay hosted payment page.
  • 3. Customer authorizes a recurring payment method.
  • 4. HitPay securely tokenizes the payment method.
  • 5. Odoo automatically stores the payment token.
  • 6. The initial subscription payment is completed.
  • 7. Future subscription renewals are charged automatically using the saved payment method.

Webhook Processing

The module automatically handles HitPay recurring billing webhooks to manage the payment method lifecycle.

  • • Automatic payment method attachment.
  • • Automatic payment method detachment.
  • • Secure webhook signature verification.
  • • Automatic webhook lifecycle management.

Recurring payment transactions are processed synchronously using the immediate API response returned by HitPay, ensuring reliable transaction processing.

Important Note

Subscription products intended for automatic recurring billing should use Invoicing Policy = Ordered Quantities. Products configured with Delivered Quantities cannot be automatically invoiced until they become invoiceable according to Odoo's standard invoicing rules.
